Dataset Structure

TransBiolab captures transparent laboratory objects using Intel RealSense cameras, providing RGB-D images with precise 6DoF ground truth annotations.

📁 Scene Directory Layout

000020/                              # Scene directory
├── rgb/                             # Color images
│   ├── 000000.png                   # Frame 0
│   ├── 000001.png                   # Frame 1
│   └── ...                          # {frame_id}.png
│
├── depth/                           # Depth maps
│   ├── 000000.png                   # Frame 0
│   └── ...                          # 1:1 with rgb/
│
├── mask/                            # Full projected masks
│   ├── 000000_000000.png            # Frame 0, Object 0
│   ├── 000000_000001.png            # Frame 0, Object 1
│   └── ...                          # {frame_id}_{obj_idx}.png
│
├── mask_visib/                      # Visible-only masks
│   ├── 000000_000000.png            # Same naming as mask/
│   └── ...
│
├── scene_camera.json                # Per-frame camera intrinsics
├── scene_gt.json                    # Per-frame GT 6DoF pose
└── scene_gt_info.json               # Per-frame visibility info
🖼️

RGB Images

1280×720 color images captured by Intel RealSense D415 camera. PNG format with lossless compression.

rgb/{frame_id}.png
📐

Depth Maps

Aligned depth maps in 16-bit PNG. Each pixel stores depth in millimeters. Transparent regions may have missing depth.

depth/{frame_id}.png
🎭

Segmentation Masks

mask/: Full projection including occluded parts. mask_visib/: Only visible portions, accounting for inter-object occlusion.

{frame_id}_{obj_idx}.png
📋

JSON Annotations

scene_gt.json: 4×4 SE(3) pose per object. scene_camera.json: Camera K matrix. scene_gt_info.json: Visibility fraction & bounding box.

BOP format compatible

scene_gt.json

{
  "0": [
    {
      "cam_R_m2c": [0.98, -0.03, 0.17, ...],  // 3×3 rotation (row-major)
      "cam_t_m2c": [-52.3, 48.1, 621.7],       // translation in mm
      "obj_id": 5                                // object category ID
    },
    ...
  ]
}

scene_camera.json

{
  "0": {
    "cam_K": [921.8, 0, 640.4,            // 3×3 intrinsic matrix
              0, 921.5, 354.3,
              0, 0, 1],
    "depth_scale": 0.1                     // depth_pixel × scale = mm
  }
}

05

Key Findings

01

Plate-type Objects Excel

Flat, plate-shaped objects (microplates, petri dishes) achieve 91-94% ADD-S AUC and >0.72 mIoU, making them ideal for both pose estimation and segmentation tasks.

02

Large Transparent Bottles Struggle

Objects like 75ml flask and 125ml flask score as low as 41-54% ADD-S AUC. The large transparent volume causes severe depth estimation errors and feature ambiguity.

03

Viewpoint Matters for Pose

FoundationPose gains +15.6% AUC when moving from shallow (22°) to steep (62°) viewing angle. Steeper views reduce transparency artifacts and improve depth signal quality.

04

SAM3 Fails on Tiny Objects

The 35mm dish (eppendorf_cap) has 0% match rate with SAM3 text prompts, highlighting the challenge of segmenting very small transparent objects with language-guided methods.

05

Symmetry Affects ADD Metric

Highly symmetric objects show 26.9% gap between ADD-S and ADD metrics. This confirms ADD-S as the more appropriate metric for rotationally symmetric transparent objects.

06

Scene Complexity ≠ Difficulty

Performance is dominated by object intrinsic properties, not scene crowdedness. Scenes with 4 objects but containing flasks score lower than 10-object scenes with mostly plates.
